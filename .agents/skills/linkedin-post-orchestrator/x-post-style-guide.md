# X Post Style Guide

Use this guide when writing `x_post.txt` for this repository.

## Default format

- Assume a premium X account with long-form support.
- Prefer a single strong post over a thread unless the user explicitly asks for a thread.
- Default target length: roughly 400 to 900 characters.
- Extended range is allowed up to 1,500 characters when the topic genuinely earns the extra space.

## Structure

- Open with a blunt first line that contains the primary keyword.
- Use 2 to 5 short paragraphs or line breaks for readability.
- Build one coherent argument, not a listicle.
- Ground the post in a concrete engineering pain, tradeoff, failure mode, or practical insight.
- End with a takeaway that feels earned, then add 2 to 4 relevant hashtags.

## Tone

- Sound lived-in, technically literate, and anti-hype.
- Favor judgment, maintenance, trust, debugging pain, and shipping pressure over trend-chasing.
- Keep the writing sharp and useful, not polished into brand-speak.
- Longer posts should add nuance, not filler.

## Avoid

- Forcing the post under legacy free-account character limits.
- Thread numbering unless explicitly requested.
- Corporate slogans, fake urgency, or generic motivational lines.
- Keyword stuffing or hashtag spam.

## Example shape

`Developer tools are getting better at writing code and worse at admitting where the real pain lives.`

`The hard part is rarely the first draft. It is the weird edge case, the stale dependency, the missing assumption, and the hour you lose proving a bug is not where the stack trace says it is.`

`That is why the useful AI tooling story is still reduced friction, not replacement. Less time spent suffering through repetition. More time spent using judgment where it actually matters.`
