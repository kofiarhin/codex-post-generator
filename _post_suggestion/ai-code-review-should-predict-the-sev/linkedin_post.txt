AI code review should predict the SEV.

We've spent years asking tools to help us write code faster. Useful, sure. But most teams are not losing sleep because typing is slow. They lose sleep because one "small" diff nudges the wrong flag, dependency, or service boundary and suddenly the incident channel starts sounding like a family group chat during a power cut.

That's why Meta's Diff Risk Score caught my attention. Risk-aware software development is a much more mature use of AI code review than another autocomplete victory lap. Scoring code change risk, surfacing likely blast radius, and pointing reviewers at the risky part of the diff is the kind of boring feature that actually helps software delivery.

The best developer tools rarely feel magical. They feel annoyingly accurate. They make production incident prevention less dependent on the one engineer who still remembers why a 2019 migration script is holding half the system together.

I don't need AI to clap when I rename a variable.
I need it to tell me which merge request is about to ruin everyone's Thursday.

If AI code review gets better at predicting risk instead of generating volume, does that change how you think about code review?
