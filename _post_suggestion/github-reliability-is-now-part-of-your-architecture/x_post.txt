GitHub reliability is no longer a background concern.

When DNS issues affect webhooks, Git operations, Actions, cloud agents, and metrics, your developer tooling starts looking a lot like production infrastructure.

Same with delayed Ubuntu runners or merge queue regressions.

The painful part is not waiting for CI.

It is realizing your release process depends on a platform you probably never modeled as a dependency.

Treat GitHub like architecture: monitor it, plan fallbacks, and make release engineering resilient to the floor moving.
