The Stack Overflow redesign debate misses the uncomfortable part: the UI was never the real problem.

Developers did not stop opening Stack Overflow because the buttons felt old. We stopped because AI coding tools turned “good enough, right now” into the default behavior. When the answer is one tab away in your editor, a community site has to offer something an autocomplete box cannot.

That thing is developer trust.

A real technical Q&A community does more than hand over a fix. It shows tradeoffs, edge cases, dissent, weird production scars, and the comment where someone says, “careful, this breaks on version 3.12.” That messy human context is still valuable. Honestly, it is more valuable now, because polished wrong answers are getting cheaper.

So yes, a Stack Overflow redesign might improve navigation and readability. Fine. But it does not solve the harder product problem: why should a tired engineer leave the flow of work to ask humans for help?

If I were building for this moment, I would obsess less over layout and more over visible expertise, stronger curation, and better ways to surface why an answer is trustworthy.

The AI era did not kill the developer community. It just removed the convenience moat.

What would make you actively return to a technical Q&A community instead of asking a model first?
